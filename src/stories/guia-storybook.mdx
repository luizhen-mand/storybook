  <h1>Guia de Criação de Componentes</h1>
  <p>Este documento serve como um guia para o desenvolvimento de componentes no <strong>Design System Mandarin</strong>, utilizando <strong>Next.js</strong>, <strong>TypeScript</strong>, <strong>CSS</strong> e <strong>Storybook</strong>. Os componentes seguem a metodologia <strong>BEM</strong> para organização de classes CSS juntamente com <b>Composition Pattern</b> para criação de componentes flexíveis e escaláveis.</p>

  <h2>Principais Tecnologias Utilizadas:</h2>
  <ul>
    <li><strong>Next.js</strong>: Framework React que permite a criação de aplicações rápidas e otimizadas.</li>
    <li><strong>TypeScript</strong>: Linguagem que adiciona tipagem estática ao JavaScript, ajudando a evitar erros durante o desenvolvimento.</li>
    <li><strong>CSS:</strong> Tecnologia confiável que não será abandonada, garantindo a sustentabilidade do projeto.</li>
    <li><strong>Storybook</strong>: Biblioteca que permite a visualização e teste dos componentes.</li>
  </ul>

  <h2>Padrão de nomenclatura das classes:</h2>
  <p>Para manter a integridade e evitar conflitos entre o nome das classes, é necessário utilizar uma nomenclatura padrão.</p>
  <ul>
    <li><strong>Prefixo</strong>: O nome deve sempre comçar com <code>ds-</code>.</li>
    <li><strong>Nome</strong>: Em seguida vem o nome do arquivo (ex: <code>button</code>, <code>card</code>).</li>
  </ul>
  <p>É importante verificar se o nome criado já existe no projeto.</p>

    <h2>Metodologia BEM (Block, Element, Modifier):</h2>
  <p><strong>BEM</strong> é uma metodologia de nomenclatura para CSS que permite uma estrutura mais organizada e previsível.</p>
  <ul>
    <li><strong>Block</strong>: O componente autônomo que representa uma parte distinta da interface (ex.: <code>.button</code>).</li>
    <li><strong>Element</strong>: Parte de um bloco que serve a uma função específica (ex.: <code>.button__icon</code>).</li>
    <li><strong>Modifier</strong>: Variação do bloco ou elemento que altera sua aparência ou comportamento (ex.: <code>.button--large</code>).</li>
  </ul>

    Exemplo de como seria o html: <br/>

    Exemplo de como seria o css: <br/>
    <Image width={650} height="auto" src={BEM2} />

    <h2>Composition pattern</h2>
    <strong>Composition Pattern</strong> é uma abordagem que promove a utilização de componentes por meio da composição. Em vez de criar componentes altamente específicos, você cria componentes mais simples e flexíveis, que podem ser combinados para formar a interface desejada, sem a necessidade de alterar o componente via props. 
    
    Por exemplo, um componente de botão seria criado da seguinte maneira:

    <b>Sem ícone</b> <br/>
    <Image width={650} height="auto" src={Component3} />

    <h2>Passos para Criação de um Componente</h2>

  <h3>1. Verifique os Componentes Existentes</h3>
  <p>Antes de criar um novo componente, veja se já existe algum componente similar no Design System. Se já estiver implementado, considere criar uma variação, utilizando o princípio de reutilização.</p>

  <h3>2. Crie um Novo Componente ou Variação</h3>

  <h4>Estrutura de Arquivos</h4>
  <p>Os componentes devem seguir a estrutura abaixo para manter a organização:</p>
  <Image width={650} height="auto" src={Pasta} />
  <br/>
  <ul>
  <li>button.css (estilos do botão)</li>
  <li>Button.stories.tsx (histórias do Storybook)</li>
  <li>index.tsx (componente Button)</li>
  <li>types.ts (tipagem do componente)</li>
  </ul>

  <h4>Importe o react em index.tsx</h4>
  <p>A importação do react se faz necessária em todos os componentes.</p>
  ```bash
  import React from 'react'
  ```
  
  <h3>3. Adicione o Componente ao Storybook</h3>
  <p>No arquivo de histórias, adicione as variações do componente, para que todas as funcionalidades e estados possam ser visualizados.</p>
  
  <h3>4. Adicione Interações para Testar o Componente</h3>
  <p>Para testar interações dentro do Storybook, utilize o <strong>addon de interações</strong> com a biblioteca <code>@storybook/test</code>. Isso permitirá a simulação de interações como cliques e mudanças de estado diretamente no Storybook.</p>

  <h3>5. Revise os Testes de Interação e Acessibilidade</h3>
  <p>Além de testar as interações, é importante garantir que o componente atenda aos padrões de acessibilidade (A11y). O Storybook vai exibir automaticamente na aba 'Accessibility' os problemas de acessibilidade nos componentes.</p>
