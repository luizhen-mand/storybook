<h1>Guia de Utilização do Design System</h1>

Este documento apresenta um guia de como clonar, atualizar e utilizar o <strong>Design System Mandarin</strong> em outros repositórios.

## Configurar o repositório localmente

- ### Clone o repositório
Utilizando o protocolo SSH, clone o repositório com o seguinte comando:
```bash
git clone git@github.com:luizhen-mand/storybook.git
```

- ### Instale as dependências
Utilize o gerenciador de dependências `npm`. Instale as dependências com o seguinte comando:
```bash
npm install
```

- ### Inicie Storybook
Para iniciar o Storybook, utilize o comando::
```bash
npm run storybook
```

## Atualizar o Design System
- ### Crie uma nova branch
Crie uma nova branch a partir da branch `main`. O nome da branch deverá seguir o seguinte padrão:
<ul style={{ listStyle: 'lower-alpha' }}>
<li>
<strong>patch/</strong>nome-da-branch <br/>
Utilizado para alterações que corrigem pequenos problemas, mas não adicionam novas funcionalidades nem alteram as existentes. Exemplos: <br/>
<strong>Correção de bug visual</strong> - Um espaçamento ou alinhamento incorreto em um componente que foi ajustado sem alterar como o componente é usado. <br/>
<strong>Ajustes de acessibilidade</strong> - Melhorias na acessibilidade (como correção de atributos aria ou contraste de cores) sem impacto no comportamento ou na estrutura dos componentes.
</li>
<br/>
<li>
<strong>minor/</strong>nome-da-branch <br />
Utilizado para adicionar novas funcionalidades de forma compatível, sem quebrar a compatibilidade com versões anteriores. Exemplos: <br/>
<strong>Novas propriedades opcionais</strong> - Adicionar novas propriedades opcionais a um componente existente, sem alterar as propriedades já existentes. <br/>
<strong>Adição de novos componentes</strong> - Adicionar um novo componente ao <strong>Design System Mandarin</strong>.
</li>
<br/>
<li>
<strong>major/</strong>nome-da-branch <br/>
Utilizado para alterações que quebram a compatibilidade com versões anteriores. Exemplos: <br/>
<strong>Remoção ou renomeação de componentes</strong> - Remover ou renomear algum componente. <br/>
<strong>Alterações no comportamento de componentes</strong> - Alterar o comportamento de um componente de forma que ele não funcione mais da mesma maneira, como mudar eventos ou propriedades obrigatórias.
</li>
</ul>

- ### Realize as modificações
Realize as modificações necessárias no código e registre as alterações utilizando o comando de commit.

- ### Abra uma PR para main
Após realizado o processo de modificações e testes, abra uma PR para fazer o merge da branch criada para a branch `main`. Após o merge em `main`, o package será automaticamente atualizado para a próxima versão e poderá ser atualizado nos projetos que utilizam o <strong>Design System Mandarin</strong>.

## Utilização

Para adicionar este package em algum projeto, siga os passos:

- ### Adicione token de autenticação 

Crie o arquivo .npmrc na raíz do projeto e adicione o token de autenticação.

- ### Instale o package

Instale o package rodando o seguinte comando:

```bash
npm i @[inserir nome]/package
```

- ### Importe o componente e o estilo

Importe os estilos dos componente com:

```bash
import '@[inserir nome]package/dist/index.css'
```

Importe o componente desejado.

- ### Atualizar versão

Caso queira atualizar a versão do package, rode o seguinte comando:

```bash
npm install
```
